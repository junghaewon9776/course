## 2. 경험치(XP) 계산 = 퀘스트 합산

XP는 **SUPER가 만든 퀘스트 규칙의 합**으로 계산된다. 퀘스트가 하나도 없으면 기본값(방역 10XP, 세트 100XP)으로 동작.

### 퀘스트 1개의 구성
| 항목 | 설명 |
|---|---|
| **이름(label)** | 표시용 이름 (예: "민원 제보") |
| **대상 행동(trigger)** | `방역 완료` / `골고루 세트` / `민원·요청 글 작성` / `민원·요청 댓글` |
| **분류(category)** | 글·댓글일 때만. `전체분류` 또는 특정 분류(거점추가요청·방역금지요청·주민민원 등) |
| **모드(mode)** | `횟수당` 또는 `N회 이상 보너스` |
| **기준횟수(threshold)** | `N회 이상 보너스`일 때만. 이 횟수 이상이면 보너스 1회 지급 |
| **XP** | 지급 점수 |

### 모드별 계산
- **횟수당(each)**: `XP × 행동 횟수`
  - 예) 댓글 1점 → 댓글 7개 = 7점
- **N회 이상 보너스(threshold)**: `행동 횟수 ≥ 기준횟수` 이면 `+XP` (1회만)
  - 예) 댓글 10회 이상 시 +50 → 댓글 12개면 +50점
  - 계단식으로 주려면 여러 개 추가: 10회↑ +50, 20회↑ +100, 30회↑ +200 …

> ⚠️ "초과"를 엄격히 하려면 기준횟수를 +1 해서 넣으면 됨(예: 10회 "초과" = 기준 11).

### 대상 행동(trigger) 정의
| trigger | 세는 대상 | 작성자 매칭 |
|---|---|---|
| `vax` 방역 완료 | 코스완료 로그 1건 = 1회 (운전+보조 모두 인정) | 로그 crew 이름 |
| `set` 골고루 세트 | 행사의 모든 코스를 고루 한 세트(코스별 최소값) | 로그 crew 이름 |
| `inquiry` 글 작성 | `data.inquiries` 글 (분류 필터 가능) | 글 `writer`(이름) |
| `comment` 댓글 | 글의 댓글 (분류 필터 가능) | 댓글 `writer`(이름) |
| `monthTop` 월별 1등 | **방역 최다**로 그 달 1등한 횟수(월 수) | 로그 crew 이름 |
| `yearTop` 년도 1등 | 선택 년도 **방역 최다** 1등이면 1 | 로그 crew 이름 |
| `photoField` 현장사진 | 코스 중 올린 현장사진(type=field) | 사진 `uploader` |
| `photoReceipt` 영수증 | 올린 영수증 사진(type=receipt) | 사진 `uploader` |
| `photo` 사진 전체 | 현장+영수증 모두 | 사진 `uploader` |
| `km` 운행거리 | 궤적 거리 합산, 1km당 1회 (내림) | 로그 crew 이름 (조 2명) |
| `time10` 운행시간 | 시작~종료 시간 합산, 10분당 1회 (내림) | 로그 crew 이름 (조 2명) |

> 사진 업로더 매칭: 로그인한 회원 이름(없으면 코스 운전자명). **이 배포 이후 올린 사진부터** 집계(기존 사진은 업로더 정보 없음).
> 사진 저장: `data.photos[photoId] = { type:'field'|'receipt', uploader, uploaderUid, takenAt, ... }`

> `monthTop`은 "1등한 달의 수"로 센다 → `횟수당`이면 3개월 1등 = XP×3. `yearTop`은 0/1.
> 월별/년도 1등의 경쟁 지표는 **방역 횟수**(명예의전당과 동일). 동점이면 먼저 계산된 1명만.

**중요:** 작성자 매칭은 **등록된 이름**으로 한다. 회원이 **자기 계정(폰번호)으로 로그인**해서 글/댓글을 써야 본인에게 XP가 쌓인다. (로그인 이름 = `memberAuth[uid]` → 회원 레코드 name)

## 3. 계급(Rank)

- XP를 **계급표(tiers)** 의 `최소XP` 구간에 매칭해서 계급 결정.
- 기본 계급 21단계: 이등병→일병→상병→병장→하사…원사→준위→소위…대위→소령…대령→준장…대장→부원수→원수.
- 계급장은 **군대식 SVG(금색 광택)**:
  - `▔` 작대기(병사) · `∨` 갈매기(부사관) · `◈`/`◆` 다이아(준위/위관) · `❖` 무궁화(영관) · `★` 별(장성)
  - 모양 칸에 이 글자를 넣으면 자동으로 계급장 그림이 나옴. 다른 글자를 넣으면 그 글자가 그대로 표시.
- **수동 계급 변경(SUPER)**: 계급표 각 사람 계급 칸의 셀렉트로 `자동(XP)` 또는 특정 계급 고정. `(수동)` 표시됨. 표시만 바뀌고 진급 푸시는 실제 XP 기준.
- **계급 커스텀 이미지(SUPER)**: 계급 설정에서 각 계급마다 🖼 버튼으로 이미지 업로드 → 그 계급은 이미지로 표시(96px 축소 dataURL로 `tier.img` 저장). ↺ 로 이미지 제거하면 다시 글자 계급장. 이미지 없으면 모양(글자→SVG) 그대로.

## 3-1. 🏅 퀘스트 훈장

- "내 계급" 카드 하단에 본인이 달성한 퀘스트가 훈장 칩으로 표시됨.
  - **N회 이상 보너스** 퀘스트: 달성 시 `🏅 이름`(금색), 미달성 시 `🔒 이름 N/기준`(진행도).
  - **횟수당** 퀘스트: `🎖 이름 ×횟수`.
- 계산은 `computeXpForName(name).items` (퀘스트별 count/done) 기준.

## 4. 진급 축하 푸시

- 코스완료(방역) 또는 민원 글/댓글 작성으로 **계급이 오르면** 자동 푸시.
  - 본인: "🎖 진급을 축하합니다! OO님이 병장 계급으로 진급했습니다! (450 XP)"
  - 전체: "🎖 진급 소식 · OO님 → 병장 진급!"
- **소급 스팸 방지**: 각 사람의 첫 기록은 "기준"만 저장하고 알림 안 감. 그 다음부터 오를 때만 축하.
- 상태 저장: `/rankState/{이름}` = 마지막 계급 인덱스.
- 푸시는 GAS 웹앱(uid 필터) 경유. `sendAppPush(title, body, target, category, targetUid)`.

## 6. SUPER 설정 방법

통계 → 계급표의 **`⚙ 계급·퀘스트 설정`** (SUPER만 보임)

1. **🧩 퀘스트**: `+ 퀘스트 추가` → 이름/대상행동/분류/모드/기준횟수/XP 입력
2. **🎖 계급**: `+ 계급 추가` → 모양/이름/최소XP 입력
3. **저장** → 전체 반영(회원 화면 포함, 앱은 재실행)

### 예시 설정
| 이름 | 행동 | 분류 | 모드 | 기준 | XP |
|---|---|---|---|---|---|
| 방역 완료 | vax | - | 횟수당 | - | 10 |
| 골고루 세트 | set | - | 횟수당 | - | 100 |
| 민원 글 | inquiry | 전체분류 | 횟수당 | - | 5 |
| 거점요청 | inquiry | 거점추가요청 | 횟수당 | - | 20 |
| 댓글 | comment | 전체분류 | 횟수당 | - | 1 |
| 댓글왕 | comment | 전체분류 | N회 이상 | 10 | 50 |

## 7. 데이터 구조 (Firebase)

```
data.quests       = [{ id, label, trigger, category, mode, threshold, xp }]
data.rankTiers    = [{ name, minXp, icon }]          // 계급 단계
data.xpConfig     = { perVax, setBonus }             // 진급엔진 호환용(vax/set 동기화)
data.rankOverride = { "이름": "계급명" }              // 수동 계급 고정
data.rankState    = { "이름": 계급인덱스 }            // 진급 감지 기준(공통: /rankState)
data.inquiries    = [{ id, category, writer, writerUid, comments:[{writer, text, createdAt}], createdAt }]
data.logs         = [{ eventId, courseId, crew:{driver, assist}, finishedAt, startedAt, ... }]
data.members      = [{ id, name, phone }]
data.memberAuth   = { uid: memberId }                // 회원 계정 → 회원
```
