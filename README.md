# 코스 관리 시스템

행사별(방역행사, 효도위안잔치 등) 코스를 등록하고, 조별 로테이션으로 운영하며, GPS로 실시간 진행 상황을 추적하는 웹앱입니다.

## 파일 구성

```
방역코스/
├── index.html       # 메인 메뉴 + 데이터 백업/복원
├── admin.html       # 관리자: 행사·코스·거점·조·스케줄 등록
├── today.html       # 기사용: 오늘의 운영 (모바일 + GPS)
├── monitor.html     # 본부용: 운행 기록 조회
├── common.js        # 공통 데이터 관리/로테이션 계산
├── style.css        # 스타일
└── README.md        # 이 파일
```

## 사전 준비

### 1. 카카오 디벨로퍼스 키 발급 (무료)

1. https://developers.kakao.com 접속 → 로그인
2. **내 애플리케이션 → 애플리케이션 추가하기**
3. 생성된 앱 클릭 → **앱 키** 메뉴
4. **JavaScript 키** 복사
5. **플랫폼 → Web 플랫폼 등록** → 사이트 도메인 추가
   - 로컬 테스트: `http://localhost`, `http://127.0.0.1`
   - 배포 후: `https://your-domain.com`

### 2. 키 적용

`common.js` 파일 두 번째 줄:
```javascript
const KAKAO_KEY = 'YOUR_KAKAO_JAVASCRIPT_KEY'; // ← 여기에 발급받은 키 붙여넣기
```

## 사용 방법

### 첫 실행 (관리자)

1. `admin.html` 열기
2. **행사** 추가 → 예: "방역행사", "효도위안잔치"
3. **코스** 추가 → 행사별로 1~5코스 자유 추가
4. **조** 추가 → 인원/차량/조장 입력
5. **스케줄** 추가 → 운영 요일 + 로테이션 순서
6. 지도에서 클릭으로 **거점 핀** 찍기 (코스 선택 후)
   - 핀은 드래그로 위치 미세조정 가능
   - 거점 순서는 ↑↓ 버튼으로 변경

### 매일 운영 (기사)

1. 모바일에서 `today.html` 열기
2. 오늘 배정된 코스 목록에서 자기 조 코스 클릭
3. **📍 GPS 시작** 버튼 탭
4. 거점 도착 시 자동 알림 → 방역 완료 처리
   - 또는 거점별 **🧭** 버튼으로 카카오내비 호출
   - **✓** 버튼으로 수동 완료 처리
5. 모든 거점 끝나면 **✅ 코스 완료 보고**

### 모니터링 (본부)

1. `monitor.html` 열기
2. 기간 선택 (오늘/주/월/전체)
3. 운행 목록에서 항목 클릭 → 지도에 궤적 표시

### 데이터 백업/복원

`index.html`의 **💾 백업/복원** → JSON 파일로 다운로드/업로드

## 로컬에서 테스트

브라우저에서 직접 HTML 파일을 열어도 동작하지만, **GPS는 HTTPS에서만 동작**합니다.

로컬 테스트용 간이 서버:
```bash
cd 방역코스
python -m http.server 8000
# 브라우저에서 http://localhost:8000
```

## 무료 배포 (HTTPS)

GPS와 카카오 API는 HTTPS 환경에서 안정적입니다. 추천:

- **Netlify**: 폴더 통째로 드래그앤드롭하면 끝
- **Vercel**: GitHub 연동 후 자동 배포
- **GitHub Pages**: GitHub 저장소에 push

배포 후 그 도메인을 카카오 디벨로퍼스 → 플랫폼에 추가해야 합니다.

## 데이터 저장 방식

현재는 **브라우저 localStorage**에 저장됩니다.
- 같은 폰/PC에서만 데이터 공유
- 다른 기기와 동기화하려면:
  1. **백업/복원**으로 JSON 주고받기 (간단)
  2. **Firebase Realtime Database** 연동 (실시간, 무료 티어 충분)

## 한계와 다음 단계

- 카카오내비 앱은 **외부 파일을 불러와 자동 주행하는 기능이 없음**
  → 거점 1개씩 URL Scheme으로 호출하는 방식 사용
- 본부 모니터링은 **localStorage 기반이라 실시간 다중 기기 동기화 불가**
  → Firebase 붙이면 해결
- 음성 턴바이턴 안내 필요 시 카카오내비 호출 방식 사용

## 데이터 구조 (참고)

```json
{
  "events": [{ "id", "name", "courses": [{ "id", "name", "color" }] }],
  "anchors": [{ "id", "name", "lat", "lng", "memo", "eventId", "courseId", "order" }],
  "teams": [{ "id", "name", "vehicle", "leader", "members": [] }],
  "schedules": [{ "eventId", "startDate", "weekdays": [1,3,5], "rotation": ["t1","t2","t3"] }],
  "logs": [{ "date", "eventId", "courseId", "teamId", "completions": [], "track": [], "startedAt", "finishedAt" }]
}
```

## 로테이션 동작 방식

`스케줄.startDate`부터 운영일을 카운트하여 `(코스 인덱스 + (운영일수-1)) % 조 수`로 배정합니다.

예: 3코스 + 3조, 월수금 운영
- 1주차 월: 1코스→1조, 2코스→2조, 3코스→3조
- 1주차 수: 1코스→2조, 2코스→3조, 3코스→1조
- 1주차 금: 1코스→3조, 2코스→1조, 3코스→2조
- 2주차 월: 다시 1코스→1조...
